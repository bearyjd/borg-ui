# src-tauri

## Purpose
Tauri 2 Rust backend for the desktop application. Defines Tauri commands that the Svelte frontend calls via IPC, initializes the app state (BorgClient), and configures plugins and capabilities.

## Key Files

| File | Description |
|------|-------------|
| `Cargo.toml` | Crate `borg-ui` — depends on `borg-core`, `borg-platform-win`, `tauri`, `tauri-plugin-shell` |
| `tauri.conf.json` | Tauri app configuration (window size, title, bundle settings) |
| `build.rs` | Tauri build script |
| `src/main.rs` | Entry point — calls `borg_ui_lib::run()`, sets `windows_subsystem` in release |
| `src/lib.rs` | App initialization: tracing setup, BorgClient creation, Tauri builder with command handlers |
| `src/commands.rs` | Tauri commands: `get_borg_version`, `test_ssh_connection`, `get_repo_info`, `list_archives`, `create_backup` |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `capabilities/` | Tauri capability definitions (`default.json`) |
| `gen/` | Auto-generated Tauri schemas — do not edit |
| `icons/` | App icons for all platforms (Windows, macOS, Android, iOS) |

## For AI Agents

### Working In This Directory
- `src/commands.rs` is the bridge between frontend and backend. Each `#[tauri::command]` maps to a frontend `invoke('command_name', {...})` call.
- `AppState` holds the `BorgClient` instance, managed via `tauri::Builder::manage()`.
- The borg binary path defaults to `borg.exe` next to the app executable.
- Commands return `Result<T, String>` — Tauri requires string errors for IPC.
- Do not edit files in `gen/` — they are auto-generated by Tauri.
- Icons in `icons/` are generated from the source `app-icon.png` at the workspace root.

### Testing Requirements
- `cargo test -p borg-ui` from workspace root
- Command handlers are thin wrappers — test business logic in `borg-core` instead

### Adding a New Command
1. Add the function in `src/commands.rs` with `#[tauri::command]`
2. Register it in `src/lib.rs` inside `tauri::generate_handler![]`
3. Call from frontend: `await invoke('command_name', { params })`

## Dependencies

### Internal
- `borg-core` — all backup logic
- `borg-platform-win` — Windows platform features

### External
- `tauri` 2.x — app framework and IPC
- `tauri-plugin-shell` 2.x — shell command execution
- `tracing-subscriber` — log output with env filter
